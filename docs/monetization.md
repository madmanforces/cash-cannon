# MONEY BIZ Monetization

## 1. 수익화 원칙

- 사용자가 `돈을 더 벌거나 덜 잃게 만든다`는 명분이 명확해야 한다.
- 기본 과금은 구독으로 가되, AI 생성과 업종 템플릿은 add-on으로 분리한다.
- 가격은 `생산성 도구`가 아니라 `매출 도구` 기준으로 설계한다.

## 2. 권장 요금제

### Free

- 오늘의 액션 1개
- 월 AI 생성 20회
- 상품/서비스 5개까지
- 단일 사업만 관리 가능

### Pro

- 월 `KRW 19,900`
- 오늘의 액션 무제한
- 월 AI 생성 500회
- 성과 리포트
- 고객 세그먼트 메시지 추천
- 업종별 템플릿 팩 기본 제공

### Team

- 월 `KRW 49,000`
- 팀원 5명
- 매장/브랜드 다중 관리
- 승인 플로우
- 팀용 리포트

### Add-on

- AI 크레딧 200회: `KRW 9,900`
- 업종 특화 템플릿 팩: `KRW 4,900 ~ 14,900`
- 시즌 캠페인 팩: `KRW 9,900`

## 3. 왜 하이브리드 모델인가

- 구독은 예측 가능한 MRR을 만든다.
- Add-on은 라이트 유저와 시즌성 수요를 흡수한다.
- AI 생성 기능은 사용량 차이가 커서 구독만으로는 손익 관리가 어렵다.

권장 구조:

- 기본 가치: 구독
- 초과 사용량/특화 가치: Add-on

## 4. 전환 퍼널 가정

아래 숫자는 설계용 가정치다.

- 방문자 10,000
- 가입 전환 20%
- 온보딩 완료 60%
- 첫 데이터 입력 65%
- 7일 내 첫 액션 실행 35%
- 무료 사용자 중 Pro 체험 시작 10%
- 체험 사용자 중 결제 전환 25%

이 경우 월간 초기 유료 사용자 수는 대략 `195명` 수준이다.

## 5. 초기 매출 가정

설계용 시나리오:

- Pro 160명 x KRW 19,900 = KRW 3,184,000
- Team 35명 x KRW 49,000 = KRW 1,715,000
- Add-on 매출 = KRW 900,000

예상 월 매출 합계:

- `KRW 5,799,000`

이 수치는 어디까지나 초기 가정이며 실제 CAC, 리텐션, AI 비용에 따라 조정해야 한다.

## 6. 핵심 실험

### 실험 A

- 무료 플랜의 AI 생성 횟수 제한을 어디에 둘지 테스트
- 목적: 무료 체험 가치는 유지하면서 Pro 전환 유도

### 실험 B

- `월 구독만`과 `구독 + AI 크레딧` 비교
- 목적: ARPPU와 이탈률 균형 확인

### 실험 C

- `매출 상승` 메시지 vs `운영 시간 절감` 메시지 비교
- 목적: 랜딩 페이지 전환율 개선

### 실험 D

- 업종별 템플릿 팩 번들 가격 테스트
- 목적: Add-on 객단가 최적화

## 7. 반드시 추적할 지표

- Trial start rate
- Trial to paid conversion
- Monthly churn
- AI request cost per paying user
- Add-on attach rate
- ARPPU
- LTV / CAC

## 8. 수익화 리스크

- 구독 가격이 낮으면 AI 사용량이 손익을 악화시킨다.
- 구독 가격이 너무 높으면 라이트 사업자가 진입하지 못한다.
- 결과가 추상적이면 결제 유지율이 떨어진다.

따라서 홈 화면은 `콘텐츠 생성`보다 `실행 가능한 액션`을 먼저 보여줘야 한다.
